# Sprint 1 — Requirements

## What Is Being Built
No code. This sprint produces two things: a thorough understanding of the starter repo data, and a fully written `planning.md`.

## Business Justification
The spec explicitly requires planning.md to be written before any implementation. It is a graded deliverable. The Builder cannot implement tools without knowing the exact listings data structure and the tool specs defined in planning.md.

## Scope
| In Scope                                              | Out of Scope                     |
|-------------------------------------------------------|----------------------------------|
| Read data/listings.json (5–10 listings)               | Any code in tools.py             |
| Read data/wardrobe_schema.json                        | Any code in agent.py             |
| Read utils/data_loader.py                             | Any code in app.py               |
| Read tools.py stubs and docstrings                    | Tests                            |
| Read agent.py structure                               | Stretch features                 |
| Write planning.md — all sections, no placeholders     |                                  |
| Write complete interaction walkthrough in planning.md |                                  |

## Deliverables
1. `planning.md` fully written — every section filled with specific, implementation-ready content
2. No code files modified

## Success Definition
planning.md describes all three tools with exact parameter names and types. The planning loop section describes specific conditional branches (not "it decides what to call"). The architecture diagram labels all components and data flows. The AI Tool Plan names specific sections to use as prompts.
